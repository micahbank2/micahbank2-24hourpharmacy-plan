1. Prescription discount card commissions
Highest potentialRecurringVerified active
$1.50-4/fill
Visitors save a free discount card. Every time they use it at a pharmacy, you earn a commission. The affiliate code is stored in the pharmacy system permanently, so one visitor generates revenue for years. This is the best revenue stream because it compounds: 100 active card users filling 2 prescriptions/month at $3/fill = $600/month from just those 100 people, growing every month as new users save the card.

Part 3: How the tech stack fits together

Where you build things

Claude Code is where all custom code is written: the child theme (PHP templates, CSS), the custom plugin (PHP), React widgets (JSX), and Python data pipeline scripts. Code lives in your GitHub repo.

WordPress admin is where you manage content after code is deployed: write city page content, add pharmacy listings, configure affiliate codes in the settings page, manage plugins, and moderate the site.

This project chat is where you plan, review, and get prompts to paste into Claude Code. Strategy, architecture decisions, and debugging happen here.

How code gets from your repo to the live site

Phase 1 (now): Manual deployment. After Claude Code builds files, you upload the child theme and plugin folders to WordPress via the admin dashboard (Appearance > Themes > Upload, Plugins > Upload) or via Hostinger's file manager / SFTP.

Phase 2 (when deploy frequency increases): Set up a GitHub Actions workflow that automatically SFTPs changed files to Hostinger whenever you push to the main branch. Claude Code can build this for you.

React widgets: Built locally with npm run build in each widget directory. The compiled JS file outputs directly into the plugin's assets folder. Upload the plugin and the widget JS goes with it.

How changes work after you're live

Content changes (editing city pages, publishing blog posts, updating pharmacy data): Done directly in WordPress admin. No code needed.

Affiliate code changes (new BIN/PCN, new program IDs): Done in WordPress admin under 24hr Pharmacy > Settings. No code needed.

Template/style changes (new page layouts, CSS tweaks, new ad zones): Built in Claude Code, then re-upload the child theme folder.

Widget changes (new features in discount card, pharmacy finder updates): Built in Claude Code, run npm run build, re-upload the plugin folder.

New city pages at scale: Run the Python data pipeline script locally. It queries Google Places API, generates content, and pushes pages to WordPress via REST API. You then review and publish in WordPress admin.

Data flow

Pharmacy data: Google Places API (Python script) -> WordPress REST API -> City/Pharmacy posts in database -> Rendered by child theme templates -> Schema markup for Google

Affiliate codes: WordPress wp_options table (entered via settings page) -> wp_localize_script passes to React -> Widget displays BIN/PCN/Group -> User saves card -> You earn commission

Analytics: GA4 tracks pageviews/events -> Search Console tracks rankings -> Affiliate dashboards track conversions -> Wave Accounting tracks revenue
